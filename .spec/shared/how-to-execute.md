# How to execute

This file discribe and define instruction fo how to execute any especification in the project.

## General rules

1. Read the `Objective` and `Technical Context` before start.
2. Execute each test in order as they are inside the group.
3. When end each task, mark the checkbox as done and register a evidence right below.
4. Never exclude a task.
5. If a tash could not be executed, register the rasion in the evidence.
6. Only consider a task done when all checkbox are checked.
7. When a task indicate a tool or automation (skill, scripts, generators) use it as the main implemntation tool.
8. If the indicated tool doesn't works or doesn't cover all needs, apply it as far as possible and register a observation in the evidence.
9. After all done run unit tests

## Evidence format

Always use the pattern:

```md
- [x] taks description
  > YYYY-MM-DD HH:MM - what was did, decisions applied, observations and deviations of the plan
```

## Main objective

These rules exists for keep the executions traceable with the min documentation as possible.
